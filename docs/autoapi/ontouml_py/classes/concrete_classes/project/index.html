ontouml_py.classes.concrete_classes.project

Module for the Project class within an OntoUML model.

The Project class extends NamedElement to include project-specific details such as bibliographic citations, keywords, and landing pages, among others, providing a comprehensive representation of a project’s metadata.

Represent an OntoUML Project, extending NamedElement with additional project-specific metadata.

This class encapsulates various aspects of a project, such as acronyms, bibliographic citations, keywords, landing pages, and more. It provides a structured way to represent and access these details. The ‘elements’ attribute, which holds the project elements, is managed through specific methods to ensure integrity and consistency of the project’s structure.

Variables:

  • acronyms (set[str]) – A set of acronyms associated with the project, aiding in its identification and reference.

  • bibliographic_citations (set[str]) – A collection of bibliographic citations that reference/are relevant to the project

  • keywords (set[str]) – Descriptive keywords that encapsulate the essence and focus areas of the project.

  • landing_pages (set[str]) – URLs pointing to web pages that provide an entry point or overview of the project.

  • languages (set[str]) – The set of languages that are used or supported within the scope of the project.

  • namespace (Optional[str]) – An optional namespace that provides a unique context for the project’s elements.

  • sources (set[str]) – A set of sources or references that have contributed information to the project.

  • access_rights (set[str]) – Information detailing the access rights and restrictions associated with the project.

  • ontology_types (set[str]) – The types of ontologies that are utilized or represented in the project.

  • themes (set[str]) – Themes or topics that are central or relevant to the project’s objectives and content.

  • license (Optional[str]) – Optional licensing information, specifying the legal usage terms of the project’s outputs.

  • contexts (set[str]) – The contexts or environments for which the project is specifically designed.

  • designed_for_task (set[str]) – A set of tasks or objectives that the project is intended to address or facilitate.

  • publisher (Optional[str]) – The entity responsible for publishing or disseminating the project, optional.

  • root_package (Optional[Package]) – The root package of the project, serving as the entry point for the project’s structure.

  • representation_style (OntologyRepresentationStyle) – The style or methodology used for representing the ontologies within the project.

__setattr__(key, value)

Override the default attribute setting behavior to include validation for ‘root_package’.

This method intercepts the setting of the ‘root_package’ attribute to ensure that the assigned package is a part of the project’s elements. If the validation fails, a ValueError is raised.

Parameters:

  • key (str) – The name of the attribute to be set.

  • value (Any) – The value to be assigned to the attribute.

Raises:

ValueError – If ‘root_package’ is set to a package not in the project’s elements.

__validate_root_package(package)

Validate if the provided package is a part of the project’s elements.

This method checks if the specified package is included in the project’s elements set. It is used to ensure that the root package is a valid and integral part of the project’s structure.

Parameters:

package (Optional[Package]) – The package to be validated as part of the project.

Raises:

ValueError – If the package is not included in the project’s elements.

add_element(element)

Add a new element to the project’s collection of elements.

This method ensures that only instances of ProjectElement or its subclasses are added to the project. It also establishes a bidirectional relationship between the project and the element by setting the element’s ‘in_project’ attribute to this project instance.

Parameters:

element (ProjectElement) – The ProjectElement to be added.

Raises:

TypeError – If the provided element is not an instance of ProjectElement.

Return type:

None

remove_element(element)

Remove an existing element from the project’s collection of elements.

This method ensures that the element to be removed is actually part of the project. It also updates the element’s ‘in_project’ attribute to None, effectively breaking the bidirectional relationship.

Parameters:

element (ProjectElement) – The ProjectElement to be removed.

Raises:

  • TypeError – If the element is not a valid ProjectElement.

  • ValueError – If the element is not part of the project.

Return type:

None
